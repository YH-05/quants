## 1. CA-Evalとは何か

CA-Evalは、**Yの競争優位性に対する判断軸をAIで再現する**ワークフローです。

具体的には、以下を自動で行います:

1. Yが書いたアナリストレポート（Markdownファイル）を読み込む
2. レポートから「競争優位性の主張」を5〜15件抽出する
3. 各主張に対して、Yの判断基準（dogma.md + 12ルール + 12パターン）を適用し、確信度（10%〜90%）を付与する
4. SEC EDGARから取得した財務データと照合し、事実確認を行う
5. 最終的に、Yが通常行う競争優位性評価に近い結果をレポートとして出力する

**一言で言えば**: 「Yの頭の中にある評価基準を、AIに教え込んで自動評価させる仕組み」です。

### 旧実装（Dify）との違い

| 項目 | 旧実装（Dify） | CA-Eval |
|------|---------------|---------|
| ナレッジベース参照 | RAG検索（検索漏れのリスクあり） | **全KB直接読み込み（検索漏れゼロ）** |
| 財務データ | 10-Kを手動アップロード | **SEC EDGARからリアルタイム取得** |
| 処理フロー | 直列10ノード | **並列処理（所要時間短縮）** |
| 精度検証 | 手動で比較 | **Phase 2データとの自動精度検証** |
| 自己批判 | なし | **T8 AI批判プロセスで過信を抑制** |

## 2. なぜ作ったのか

### 解決したい課題

1. **Yの判断基準の明文化**: Yの評価基準はPhase 2（5銘柄のスコアリング + フィードバック）で抽出済みだが、これを体系的に活用できていなかった
2. **評価の一貫性**: 同じ基準を毎回ブレなく適用する仕組みが必要
3. **スケール**: 手動では時間がかかる評価プロセスを、レポート投入から最終出力まで自動化
4. **「既存判断への警鐘」機能**: dogma.mdセクション5.2でYが最も期待する機能 ── 既存判断に縛られていることへのAIからの指摘

### 想定する使い方

```
Yがアナリストレポートを書く
    ↓
CA-Evalに投入（コマンド1つで実行）
    ↓
AIが評価レポートを生成（各主張に確信度付き）
    ↓
Yがフィードバック記入欄に評価を記入
    ↓
フィードバックをKBに反映 → 次回以降の精度が向上
```

## 3. ワークフローの全体像

### 全体フロー図

```
Phase 0: セットアップ
  │  リサーチID生成、ディレクトリ作成、パラメータ確認
  ▼
Phase 1: データ収集（2タスク並列）
  ┌──────────────┬──────────────┐
  │ T1: SEC EDGAR│ T2: レポート  │
  │ 財務データ取得│ 構造化・解析  │
  └──────┬───────┴──────┬───────┘
         ▼              ▼
Phase 2: 主張抽出 + ルール適用
  │  T4: KB1（8ルール）+ KB3（5銘柄の実例）+ dogma.md を参照し
  │      レポートから競争優位性の主張を5-15件抽出、確信度を付与
  ▼
Phase 3: 検証（2タスク並列）
  ┌──────────────┬──────────────┐
  │ T5: 事実確認  │ T6: パターン  │
  │ SEC EDGARデータ│ KB2（12パターン）│
  │ との照合      │ との照合       │
  └──────┬───────┴──────┬───────┘
         ▼              ▼
Phase 4: レポート生成 + AI批判
  │  T7: ドラフトレポート生成
  │  T8: AI自己批判（過信チェック）→ 最終版レポート生成
  │  T9: Phase 2データとの精度検証（該当銘柄のみ）
  ▼
Phase 5: 完了
  │  最終レポート（revised-report.md）をYに提示
```

### 各タスクの役割

| タスク | 名前 | やっていること | 所要時間目安 |
|--------|------|--------------|-------------|
| T1 | SEC EDGAR取得 | 5年分の財務データ、10-K/10-Q/8-Kテキスト、インサイダー取引を取得 | 2-3分 |
| T2 | レポート解析 | Yのアナリストレポートをセクション分割し、優位性候補を抽出 | 1-2分 |
| T4 | 主張抽出 | KB1ルール + KB3実例 + dogma.md を参照し、主張を抽出・評価 | 3-5分 |
| T5 | 事実確認 | 主張内の数値・事実をSEC EDGARデータと照合 | 2-3分 |
| T6 | パターン検証 | KB2の却下パターン（A-G）/ 高評価パターン（I-V）と照合 | 1-2分 |
| T7 | レポート生成 | 全結果を統合してドラフトレポートを生成 | 2-3分 |
| T8 | AI批判 | ドラフトを自己批判し、過信を修正して最終版を生成 | 3-5分 |

**1銘柄あたりの合計**: 約15-25分

## 4. ナレッジベース（KB）の構成

CA-Evalの精度を決定するのは、Yの判断基準を記述した以下の4つのナレッジベースです。

### 4.1 dogma.md（投資判断軸）

**Phase 2の5銘柄（CHD, COST, LLY, MNST, ORLY）のフィードバックから抽出したYの評価基準。**

含まれる内容:
- **基本哲学（5原則）**: 「優位性は能力・仕組みであり結果ではない」等
- **確信度スケール（5段階）**: 90%（かなり納得）〜 10%（却下）
- **評価を上げる/下げる要因**: 具体例付き
- **CAGR接続の評価基準**: 高評価/低評価パターン
- **ブランド力評価フレームワーク**: 4類型

> **現在のステータス**: 運用版ドラフト。Yの ○/×/△ 回収後に確定予定。

### 4.2 KB1: 主張抽出ルール集（8ファイル）

Yの12の判断ルールを個別にファイル化したもの。各主張に対して全ルールを適用し、PASS / PARTIAL / FAIL を判定します。

| ルール | 内容 | 効果 |
|--------|------|------|
| ルール1 | 能力・仕組みか、結果・実績か | FAILなら確信度低下 |
| ルール2 | 名詞で表現される属性か | FAILなら確信度低下 |
| ルール4 | 定量的裏付けがあるか | PASSなら+20%程度 |
| ルール6 | 構造的要素か補完的要素か | 補完的なら注記 |
| ルール7 | 純粋競合への差別化があるか | FAILなら-10〜20% |
| ルール8 | 戦略 ≠ 優位性 | FAILなら確信度低下 |
| ルール10 | ネガティブケース（断念例）の裏付け | PASSなら+10〜20% |
| ルール11 | 業界構造と企業ポジションの合致 | PASSなら最大+30%（90%の条件） |

> ルール3（相対性）、ルール5（CAGR直接性）、ルール9（事実誤認）、ルール12（①主②従）は、dogma.md本体に統合またはPoC省略。

### 4.3 KB2: パターン集（12ファイル）

過去の評価で繰り返し出現した「よくある間違い」と「高く評価すべきパターン」を集約。

**却下パターン（確信度低下）**:

| パターン | 名前 | 典型例 |
|---------|------|--------|
| A | 結果を原因と取り違え | 「シェアが高い＝優位性」→ シェアは結果 |
| B | 業界共通で差別化にならない | 「グローバルネットワーク」→ 大手なら皆持っている |
| C | 因果関係の飛躍 | 3段階以上の推論チェーン |
| D | 定性的で定量的裏付けなし | 測定不能な曖昧な差別化 |
| E | 事実誤認 | OPMをGPMと誤認 → 即10% |
| F | 戦略を優位性と混同 | 戦略的意思決定 ≠ 構造的力 |
| G | 純粋競合との優位性不明 | 「誰と比べて？」が不明 |

**高評価パターン（確信度上昇）**:

| パターン | 名前 | 典型例 |
|---------|------|--------|
| I | 定量的裏付けのある差別化 | 退職率9% vs 業界20%+ |
| II | 直接的なCAGR接続メカニズム | 会員ミックス改善→ARPU→既存店売上 |
| III | 能力 > 結果（プロセス評価） | 買収基準の明確化、人員投入プロセス |
| IV | 構造的な市場ポジション | フラグメント市場 × 規模・密度 → 90% |
| V | 競合との具体的比較 | AZO対比でのDIFM補充頻度 |

### 4.4 KB3: few-shot例集（5ファイル）

**Phase 2でYが実際に評価した5銘柄の全主張・確信度・コメント。** AIがこれを参照することで、Yの「感覚」を学習する最も重要なデータ。

| 銘柄 | 主張数 | 平均確信度 | 特徴 |
|------|--------|-----------|------|
| ORLY | 7 | 63% | 最高評価。フラグメント市場×規模で90%が2件 |
| CHD | 7 | 50% | M&A能力・ポート管理能力が評価対象 |
| LLY | 7 | 47% | 製薬。グローバルネットワークは業界共通で30% |
| MNST | 7 | 40% | ブランドは結果の側面。因果飛躍に注意 |
| COST | 6 | 39% | 最低評価。劇場的価値は10%（定量評価不能） |

**確信度の出現分布（全34件のY実績）**:

| 確信度 | 出現率 | 件数 |
|--------|--------|------|
| 90%（かなり納得） | 6% | 2/34件 |
| 70%（おおむね納得） | 26% | 8/34件 |
| 50%（まあ納得） | **35%** | 11/34件（最頻値） |
| 30%（あまり納得しない） | 26% | 8/34件 |
| 10%（却下） | 6% | 2/34件 |

→ この分布からAIの出力が大きく乖離していないかをチェックしている。

## 5. 出力レポートの構成

1銘柄につき以下のファイルが生成されます。Yに渡す最終版は **revised-report.md** です。

### 最終レポート（revised-report.md）の構成

| セクション | 内容 |
|-----------|------|
| **1. 銘柄概要** | 事業概要、戦略、CAGR推定、CAGR内訳表 |
| **2. 競争優位性評価サマリー** | 全主張の一覧表（主張名・確信度・根拠・課題）、確信度分布、ファクトチェック結果 |
| **3. 主張別詳細評価** | 主張ごとに: AI評価コメント / 根拠（レポート引用）/ 12ルール適用マトリクス / CAGR接続評価 / 検証結果 |
| **4. 総合評価** | 強み/弱みの分析、KB3（5銘柄）との比較 |
| **5. フィードバック記入欄** | Yが各主張に対してCA確信度・CAGR確信度・コメントを記入するテンプレート |

### 各主張の評価形式（例: MCO #1）

```
主張 #1: 格付け市場の寡占構造（MCO+SPGI=80%、20年間不変）

AI評価: かなり納得（90%）

[AI批判の注記があればここに表示]

根拠（レポート引用）:
- 「MCOとSPグローバルで4割ずつのシェア...」

ルール適用結果（全12ルール）:
| ルール | 判定 | 根拠 |
|--------|------|------|
| ルール1 | PARTIAL | 寡占構造は「結果」的だが維持要因は構造的 |
| ルール11 | PASS | ORLY#2に匹敵する業界構造合致 |
| ...

CAGR接続:
| パラメータ | 確信度 | コメント |
|-----------|--------|---------|
| MIS売上+6% | 90% | 寡占→市場成長の安定享受 |

─────────────────────────────
納得度:  10% / 30% / 50% / 70% / 90%  ← 丸をつける
```

### フィードバック記入欄（例: MCO）

```
| 主張 | AI評価 | アナリスト評価（CA） | アナリスト評価（CAGR） | コメント |
|------|--------|------------|-------------|---------|
| #1 格付け市場の寡占構造 | 90% | ( ) | ( ) | |
| #2 顧客スイッチングコスト | 70% | ( ) | ( ) | |
| ...

追加フィードバック欄:
- 純粋競合の指定
- 主張の統合・削除の提案
- CAGR推定の妥当性に関する見解
```

## 7. 現在の限界と課題

### PoC段階でスキップしている機能

| 機能 | 状況 | 影響 |
|------|------|------|
| ルール12（①主②従） | スキップ | 期初レポート vs 四半期レビューの区別なし |
| 業界データ（T3） | スキップ | SEC EDGAR以外の外部データを使っていない |
| 対面フィードバックループ | 未実装 | フィードバックのKB反映は手動 |

### 精度に関する課題

1. **KB3の銘柄数が少ない（5銘柄のみ）**: AIが参照できるYの実例が34件の主張評価に限られる。新しい業界や事業モデルに対する汎化が不十分
2. **dogma.mdが未確定**: Yの ○/×/△ による検証が未実施
3. **Phase 2データの不足**: 精度検証できる銘柄がCHD, COST, LLY, MNST, ORLYの5銘柄のみ
4. **純粋競合の定義が銘柄ごとに曖昧**: ルール7（純粋競合への差別化）の適用精度が低い
5. **「共有優位性」の扱い**: MCO #1のように、SPGIとの共有優位性をどう評価するかの基準が未確立

## 8. お願いしたい協力事項

精度向上のために、以下の5つの協力をお願いしたいと考えています。優先度の高いものから記載します。

### 協力1: dogma.md の検証・確定（最優先）

**現状**: dogma.md はPhase 2のデータから抽出した「ドラフト」状態です。

**お願い**: dogma.md の各項目について ○（正しい）/ ×（違う）/ △（条件付き）で検証してください。

特に確認したい項目:
- 基本哲学（5原則）: 過不足はないか
- 確信度スケール: 5段階の定義は適切か
- 評価を上げる/下げる要因: 抜けている要因はないか
- ブランド力評価フレームワーク（4類型）: 実際の判断と合致するか

**所要時間の目安**: 30-60分

### 協力2: 生成済みレポートへのフィードバック記入

**現状**: 12銘柄分のレポートが生成済みですが、Yの評価が入っていません。

**お願い**: 各レポート末尾の「フィードバック記入欄」に以下を記入してください。

1. **各主張のCA確信度**: 10% / 30% / 50% / 70% / 90% から選択
2. **各主張のCAGR接続確信度**: 同上
3. **コメント**: AIの評価と異なる場合、その理由（特に重要）
4. **追加フィードバック**: 純粋競合の指定、主張の統合・削除の提案

**このフィードバックが最も価値が高い理由**:
- AIの評価とYの評価のギャップが、KBの改善ポイントを教えてくれる
- 特に「AIが70%と評価したがYは30%」のようなケースから、KBに欠けているルールやパターンが発見できる

**所要時間の目安**: 1銘柄あたり15-30分（まずはMCO, NFLX等のKB3に含まれない銘柄から）

### 協力3: KB3（few-shot例）の拡充

**現状**: KB3は5銘柄（CHD, COST, LLY, MNST, ORLY）のみ。

**お願い**: 協力2のフィードバックをもとに、以下の銘柄をKB3に追加したい。

優先的に追加したい銘柄の方向性:
- **KB3にない業界**: テクノロジー（MSFT）、格付け（MCO）、エンタメ（NFLX）など
- **KB3にない事業モデル**: プラットフォーム型、サブスクリプション型など
- **評価が分かれやすい銘柄**: AIとYの評価が大きく乖離した銘柄

**具体的な作業**: 協力2のフィードバック記入と同時に進められます。フィードバックの質が高い銘柄をKB3に昇格します。

### 協力4: 新規アナリストレポートの提供

**現状**: `analyst/raw/` に12銘柄分のレポートがあります。

**お願い**: 新しい銘柄のアナリストレポートを追加する際に、以下の点を意識していただけると助かります。

- **競争優位性セクション**: 優位性の記述をできるだけ明示的に含める
- **CAGR内訳**: CAGR推定値とその構成要素（売上成長、マージン改善、自社株買い等）
- **純粋競合の明記**: 「vs [競合名]」の形で比較対象を記載
- **定量データの引用**: 可能な限り数値・比率を記載

### 協力5: ルール・パターンの追加提案

**現状**: KB1（8ルール）、KB2（12パターン）は Phase 2の5銘柄から抽出。

**お願い**: 新しい銘柄のレビューを通じて、以下を発見した場合にお知らせください。

- **KB1に追加すべきルール**: 既存ルールで対応できない新しい判断基準
- **KB2に追加すべきパターン**: 繰り返し出現する「よくある間違い」や「高く評価すべきパターン」
- **既存ルール/パターンの修正**: 「この表現は自分の判断と違う」という箇所

## 9. 協力の進め方（提案）

### Step 1: dogma.md の検証（1回、30-60分）

dogma.md を読んで ○/×/△ を付けるだけで完了。これが全体の基盤になるので最初にお願いしたい。
