# 議論メモ: JSAI 2026 全34論文のNeo4j KG投入

**日付**: 2026-03-19
**議論ID**: disc-2026-03-19-jsai2026-kg-import
**参加**: ユーザー + AI

## 背景・コンテキスト

`/Volumes/personal_folder/Papers/人工知能学会/2026/` に保存されたJSAI 2026 SIG-FIN-036の全34論文PDFをNeo4j KG（v2.1スキーマ）に投入する作業。

## 実施手順

### Phase 1: 準備
- KGスキーマ（v2.1: 14ノード・29リレーション）を確認
- 既存Neo4jノードを取得（Method 44件, Anomaly 6件, MarketRegime 6件, DataRequirement 8件, Topic 12件）
- `.tmp/graph-queue/jsai2026/` ディレクトリを作成

### Phase 2: PDF読み取り・構造化抽出
- 34論文を6バッチに分割（Batch 1-5: 各6本、Batch 6: 4本）
- 6つの並列エージェントを起動し、各バッチのPDFを読み取り
- graph-queue JSON形式で構造化データを出力（合計229KB）
- 全バッチ完了: 約5分

### Phase 3: データ統合・正規化
- 6つのJSONファイルを統合
- **問題発見**: Batch 3-6のリレーションが `from`/`to` キー（Batch 1-2は `from_id`/`to_id`）
  - 71件のリレーションが欠落（空のfrom_id/to_idにマッピング）
  - 正規化ロジックで `from`→`from_id`, `to`→`to_id` に変換し修正
- ノードの重複除去: ID基準

### Phase 4: Neo4j投入
- `neo4j` Pythonドライバー（v6.1.0）をインストール
- UNWINDバッチCypherで効率的投入（21クエリで完了）
- 処理済みJSONを `.processed/` に移動
