## 2. MASF

### 設計思想

- 運用会社やヘッジファンドの投資チームをマルチAIエージェントシステム（エージェントチームメイト）で再現する
- Claude Codeの使用を前提とし、エージェントを定義するよりもスキルで役割を定義し、それをスポーンさせる形でMASを実装する。つまり、MASFは表面上はマルチエージェントシステムだが、実際はagent skillsセットである。
- どのようなスキルを組み合わせてMASFを作成するかは、コンセプト（投資ユニバースや投資戦略）次第。

### 取り入れたいアイディア

#### エージェントのスキルセット定義

ai-hedge-fund の各投資家エージェントは、暗黙的に複数のスキルを持っている。例えば Buffett Agent は「モート評価」「安全域計算」「経営陣の質の判断」を同時に行う。しかし、これらのスキルが system prompt に混在しているため、個別のスキルの品質管理が困難である。

MAS4InvestmentTeam のメモに記載された設計思想「MASFは表面上はマルチエージェントシステムだが、実際は agent skills セット」は、この問題に対する適切な回答である。各エージェントのスキルを明示的に定義し、スキルの組み合わせによってエージェントの行動を制御する方式は、以下のメリットを持つ。

- **スキルの再利用**: 「モート評価スキル」を Fundamental Analyst と CA Evaluator の両方で共有できる。
- **スキル品質の個別テスト**: 各スキルを独立にテスト・改善できる。
- **柔軟なエージェント構成**: スキルの組み合わせを変えることで、異なるコンセプトの MAS を迅速に構築できる。

#### Sentiment Agent の分離

ai-hedge-fund の Sentiment Agent は市場センチメントの分析に特化している。当プロジェクトの Phase 2 における Sentiment Analyst（T4）も同様の役割を持つが、KB 統合設計書（2026-02-25_analyst-kb-mas-integration.md）では KB1-T/KB3-T の注入先としても位置づけられている。

ai-hedge-fund からの示唆として、Sentiment Agent の役割を「市場センチメント分析」と「トランスクリプト分析」に分離することが考えられる。市場センチメント（ニュース、ソーシャルメディア）の分析は定量的・機械的処理が主であるが、トランスクリプト分析は KB-T を参照した定性的判断が必要であり、同一エージェントに両方を任せると責務が過大になる。

### 取り入れるべきではないアイディア

#### フラットなエージェント並列構造（例：viratt/ai-hedge-fund）

ai-hedge-fund の「17 エージェントが全て並列に分析し、Portfolio Manager が一括統合する」構造は、パイプラインの設計としてはシンプルだが、以下の問題がある。

- **情報のフィルタリングが不十分**: スクリーニングなしで全銘柄を 17 エージェントが分析するため、LLM コールの効率が悪い。
- **文脈の積み上げがない**: Phase 2 の分析結果を Phase 3 のディベートで深掘りし、Phase 4 で最終判断するという「段階的な理解の深化」が起きない。
- **Portfolio Manager への過負荷**: 17 エージェントの出力を同時に処理する PM のコンテキスト量が膨大。

#### 単純な多数決/平均化による判断統合（例：viratt/ai-hedge-fund）

ai-hedge-fund の Portfolio Manager は、17 エージェントのシグナル（BUY/SELL/HOLD + confidence）を「集約」して最終判断を下す。この集約方法が単純な多数決や加重平均に近い場合、以下の問題が生じる。

- **コンセンサスバイアス**: 多数決は安全で平凡な選択に収束しやすい。
- **少数意見の軽視**: Burry Agent のようなコントラリアンの鋭い洞察が多数の凡庸な BUY に埋もれる。
