# マルチエージェント金融システムの学術論文サーベイ

**作成日**: 2026-02-25
**目的**: マルチエージェントシステム（MAS）を金融・投資領域に適用した主要学術論文を体系的にサーベイし、MAS4InvestmentTeam プロジェクトの設計判断に対する学術的根拠を整理する。

---

## 1. サーベイの全体像と研究動向

### 1.1 研究領域の位置づけ

マルチエージェントシステム（Multi-Agent System: MAS）を金融市場に適用する研究は、2023年後半のGPT-4リリース以降、急速に拡大している。この流れは、従来のクオンツ投資が構造化データ（株価、財務指標）に依存していたのに対し、LLM の登場により非構造化データ（SEC Filings のテキスト、決算トランスクリプト、ニュース記事）の定量的処理が実用レベルに到達したことに起因する。

ACM CACM Blog の "Leveraging AI Multi-Agent Systems in Financial Analysis" が指摘するように、MAS の金融応用における強みは **分散インテリジェンスと分散問題解決** にある。単一の LLM がすべてを判断するのではなく、専門化されたエージェントがそれぞれの領域（ファンダメンタル分析、センチメント分析、バリュエーション）でタスクを遂行し、その結果を統合することで、単一エージェントでは到達できない判断品質を実現する。これはファンダメンタル分析に特に適しており、多様な情報タイプ（財務データ、テキスト情報、市場センチメント）の統合が本質的に求められるためである。

### 1.2 研究の時系列的発展

マルチエージェント金融システムの研究は、以下の3つのフェーズで発展してきた。

**第1フェーズ（2023年前半〜2023年後半）: 概念実証期**。TradingGPT（Li et al. 2023）や Alpha-GPT（Wang et al. 2023）が先駆けとなり、LLM を金融意思決定に活用する基本的なアーキテクチャが提案された。この時期の研究は主に単一 LLM のプロンプトエンジニアリングや、Human-in-the-Loop による半自動的なアルファ生成に焦点を当てていた。

**第2フェーズ（2024年）: アーキテクチャ分化期**。FINCON（NeurIPS 2024）、AlphaAgents（BlackRock系）、MASFIN、StockBench、HedgeAgents など、多数の論文が発表され、MAS のアーキテクチャが多様化した。パイプライン型（MASFIN）、対話型（AlphaAgents）、内部コンテスト型（Multi-Agent Trading System Based on Internal Contest Mechanism）など、エージェント間の協調プロトコルの設計空間が広がった。同時に、StockBench のようにバックテストにおける先読みバイアスや LLM のデータ汚染に対する問題提起も行われた。

**第3フェーズ（2025年〜現在）: 実用化・統合期**。Joshi (2025) の "Architectures and Challenges of AI Multi-Agent Frameworks" が示すように、現在は個別フレームワークの比較検証と、ハイブリッドアーキテクチャの探索が主たる研究課題となっている。CrewAI の協調性 + LangChain のツール柔軟性 + FinRobot のドメイン特化という組み合わせが将来の方向性として示唆されており、パフォーマンスベンチマーク・規制コンプライアンスの gap が今後の研究課題として指摘されている。

### 1.3 主要な研究軸

サーベイ対象の論文群は、以下の5つの研究軸に分類できる。

| 研究軸 | 代表的論文 | 核心的な問い |
|--------|-----------|-------------|
| **パイプライン設計** | MASFIN, HKUST | エージェントをどの順序でどう接続するか |
| **合意形成メカニズム** | AlphaAgents, FINCON | エージェント間の意見の不一致をどう解決するか |
| **アルファ生成の自動化** | Alpha-GPT, HKUST | LLM でアルファファクターを自動的に発見できるか |
| **バイアス排除** | StockBench, MASFIN | LLM の学習データ汚染と先読みバイアスをどう防ぐか |
| **リスク管理** | HedgeAgents, TradingGPT | エージェントにどうリスク意識を組み込むか |

## 2. 主要論文の詳細分析

### 2.1 MASFIN: Multi-Agent System for Decomposed Financial Reasoning and Forecasting

**著者・機関**: Rochester Institute of Technology
**核心的貢献**: シーケンシャル（順次型）パイプラインによる金融推論の分解

MASFIN は、投資判断のプロセスを5つの明確なステージに分解するパイプラインアーキテクチャを提案した。5段階は Postmortem（事後分析）、Screening（スクリーニング）、Analysis（詳細分析）、Timing（タイミング判断）、Portfolio（ポートフォリオ構築）で構成される。

最も注目すべきは **8週間のライブ運用で 7.33% の累積リターン**を達成し、同期間の S&P 500（4.92%）および NASDAQ-100 を凌駕した点である。週次リバランスを採用し、各ステージの出力が次のステージの入力となる明確なデータフローを持つ。

MASFIN の設計上の特筆点は **Postmortem Crew（事後検証班）** の存在である。上場廃止企業（NKLA, BBBYQ, REV 等）の倒産プロセスを分析し、「失敗のパターン」を抽出してスクリーニングのネガティブフィルターとして使用する。これは生存者バイアス（Survivorship Bias）の排除を構造的に保証するメカニズムであり、バックテストの妥当性を大幅に向上させる。

**限界**: 週次リバランスはターンオーバーコストが高く、実運用ではスリッページの影響が無視できない。また、ライブ運用期間が8週間と短く、統計的有意性の検証には不十分である。

### 2.2 AlphaAgents / BlackRock 系

**核心的貢献**: グループチャット（対話型）議論フレームワークと全会一致型 consensus 形成

AlphaAgents は、3つの専門エージェント（Fundamental Agent, Sentiment Agent, Valuation Agent）がグループチャット形式で対話し、全会一致で投資判断に到達するフレームワークを提案した。BlackRock の運用哲学を反映し、複数の視点からの議論を通じてハルシネーションを抑制し、より堅牢な判断を導出することを目指す。

AlphaAgents の H1 2024 における実験結果は教訓的である。**Risk-averse（リスク回避型）エージェントは、上昇相場においてボラティリティの高いテクノロジー銘柄を除外したため、ベンチマークを下回った。** この結果は「失敗」ではなく「意図したリスク特性の反映」と解釈すべきであり、MAS 運用における透明性の重要性を示している。

**設計上の重要な論点**: 全会一致型の consensus 形成は、安全だが凡庸な判断に収束する傾向がある。これは投資の文脈では「リスク回避バイアス」として現れ、コントラリアンなポジションの取得を困難にする。当プロジェクト（MAS4InvestmentTeam）がマネージャー決定型を採用した主要な理由はここにある。

### 2.3 FINCON: Synthesized LLM Multi-Agent System with Conceptual Verbal Reinforcement

**著者・機関**: NeurIPS 2024 採択
**核心的貢献**: 概念的言語強化（Conceptual Verbal Reinforcement: CVR）

FINCON は NeurIPS 2024 に採択された論文であり、エージェント間のフィードバックループを **概念的言語強化** として定式化した。従来の MAS がエージェント間の情報伝達を単純なテキスト入出力として扱っていたのに対し、FINCON はフィードバックを「概念レベル」で構造化し、過去の判断結果から学習して次の意思決定を改善するメカニズムを導入した。

CVR の核心は、エージェントが過去の投資判断の「成功要因」と「失敗要因」を言語的な概念として抽出・蓄積し、これを新しい判断の際のコンテキストとして活用する点にある。これは数値的な強化学習とは異なり、LLM の自然言語処理能力を活かした「言語的な報酬信号」と言える。

**限界**: CVR の効果を発揮するには大規模なトレーニングデータ（過去の判断事例の蓄積）が必要であり、コールドスタート問題がある。また、概念レベルのフィードバックが実際にどの程度パフォーマンスに寄与するかの因果的な検証は限定的である。

### 2.4 StockBench: Can LLM Agents Trade Stocks Profitably In Real-world Markets?

**著者・機関**: Tsinghua University
**核心的貢献**: LLM エージェントの実市場パフォーマンスのベンチマークと汚染チェック手法

StockBench は、LLM エージェントが実際の市場で利益を上げられるかを体系的に評価するベンチマークフレームワークである。最も重要な貢献は **汚染チェック（Contamination Check）** の方法論である。

汚染チェックの手順は以下の通りである。(1) LLM に特定期間の株価推移を「言い当てさせる」テストを実施する。(2) 正解率が統計的に有意に高い期間を特定する。(3) その期間は LLM の学習データに含まれている（汚染されている）と判定し、評価対象から除外する。

この手法は、LLM を用いたバックテストにおいて学習データの時間的境界が曖昧である問題に対する実用的な解決策を提供する。単にカットオフ日を設定するだけでは不十分であり、LLM が実際に「何を知っているか」を経験的に検証する必要があることを示した。

**当プロジェクトへの直接的示唆**: MAS4InvestmentTeam の3層バイアス防御（PoiT データ管理 + 時間的制約プロンプト + ティッカー匿名化）は StockBench の知見を直接的に取り入れたものである。特にティッカー匿名化は、LLM が企業名からその後の運命を推測することを防ぐ物理的な遮断メカニズムとして機能する。

### 2.5 TradingGPT (Li et al. 2023)

**核心的貢献**: 階層的メモリとエージェントの個別性格付与

TradingGPT は、マルチエージェントに **Layered Memory（階層的メモリ）** を導入した初期の研究である。メモリは短期（直近の取引結果と市場反応）、中期（数週間のトレンドとパターン認識）、長期（投資哲学とマクロ的な教訓）の3層で構成される。

さらに、**Distinct Characters（個別の性格）** として、各エージェントに異なるリスク選好や投資スタイルを付与する設計を採用した。例えば、あるエージェントはグロース志向で攻撃的、別のエージェントはバリュー志向で保守的といった性格分化を行うことで、エージェント間の議論の多様性を確保する。

**設計上の意義**: 階層的メモリは、エージェントが過去の判断を「忘れない」だけでなく、時間スケールに応じた異なる抽象度で過去の経験を保持・参照できるメカニズムである。これは人間のファンドマネージャーが持つ「経験による直感」を構造的に模倣する試みと解釈できる。

### 2.6 Alpha-GPT / Alpha-GPT 2.0 (Wang et al. 2023, 2024)

**核心的貢献**: Human-in-the-Loop によるアルファマイニングの自動化

Alpha-GPT は、人間の投資アイデア（「この業界では規模の経済が効きやすい」等の直感的な洞察）を LLM が翻訳し、定量化可能なアルファファクターに変換するフレームワークを提案した。Man Group の AlphaGPT の学術的基盤でもある。

Alpha-GPT 2.0（2024年）では、Human-in-the-Loop のプロセスをさらに洗練し、人間が提供する「シードアイデア」から LLM が複数のファクター候補を生成し、バックテストで検証するサイクルを自動化した。このサイクルを繰り返すことで、人間単独では発見できないアルファシグナルの探索を可能にする。

**学術的位置づけ**: Alpha-GPT は「完全自動化」ではなく「人間とAIの協働によるアルファ生成」を志向しており、この思想は MAS4InvestmentTeam の KB（ナレッジベース）アプローチと共鳴する。人間の投資哲学（dogma.md）を AI に注入し、その制約の下でAIが分析を行うという当プロジェクトの設計は、Alpha-GPT の Human-in-the-Loop を「オフライン化」したものと位置づけられる。

### 2.7 HedgeAgents: A Balanced-aware Multi-agent Financial Trading System

**著者・機関**: South China University of Technology
**核心的貢献**: バランス認識型マルチエージェントとエクスポージャーの動的管理

HedgeAgents は、リスク管理に焦点を当てたマルチエージェントシステムである。多くの MAS 研究がリターンの最大化に注力する中、HedgeAgents はポートフォリオのエクスポージャー（セクター偏り、ファクター偏り、集中度）を動的に監視・調整するメカニズムを中心に据えている。

「バランス認識（Balanced-aware）」とは、エージェントが個々の銘柄のアルファだけでなく、ポートフォリオ全体のリスクプロファイルを常に意識して行動することを意味する。具体的には、特定のセクターへの過度な集中が検出された場合、リスクマネージャーエージェントが他のエージェントの提案を修正・拒否する権限を持つ。

**当プロジェクトへの反映**: MAS4InvestmentTeam の Phase 5（Portfolio Construction）における制約条件（最大単一ポジション 5%、セクター上限 25%、15-30銘柄）は、HedgeAgents のバランス認識型設計から着想を得ている。

### 2.8 Automate Strategy Finding with LLM in Quant Investment (HKUST)

**著者・機関**: Hong Kong University of Science and Technology
**核心的貢献**: Seed Alpha の自動生成と Dynamic Strategy Optimization

HKUST の研究は、LLM を用いてアルファファクター（Seed Alpha）を自動生成し、これを Cross-Section Operators と Time-Series Operators の組み合わせで表現するフレームワークを提案した。生成されたファクターは **3層 MLP（多層パーセプトロン）による Dynamic Strategy Optimization** を通じて最終的なポートフォリオウェイトに変換される。

実証結果は極めて印象的である。**SSE50（上海証券取引所50指数）において、2023年1月から2024年1月の期間で 53.17% の累積リターンを達成**した。同期間の市場平均は -11.73% であり、約65ポイントの超過リターンを記録している。

HKUST の手法が注目に値するのは、LLM の出力を「最終的な売買判断」ではなく「ファクター候補の生成」に限定している点である。生成されたファクターの有効性は伝統的な統計的検証（IC 測定、バックテスト）で評価され、LLM のハルシネーションリスクを最小化している。

**3層 MLP の役割**: 10個の ReLU 活性化ノードを持つ隠れ層を備えた3層 MLP は、エージェントが生成したシグナルを市場環境に適応したウェイトに変換する「Weight Gating」として機能する。これは市場レジーム（強気・弱気・中立）に応じてファクターの有効性が変動する現実を反映した設計である。

### 2.9 その他の重要な研究

#### QuantAgent (Wang, Yuan, Ni 2024)
実世界のテスト結果をフィードバックとしてナレッジを精製する **リサーチループの自動化** を提案。バックテスト結果からエージェントが自身の知識を更新するサイクルは、当プロジェクトの Stage 5（バックテスト + 暗黙知拡充ループ）の設計に影響を与えている。

#### Multi-Agent Trading System Based on Internal Contest Mechanism (2025)
エージェント間の **内部コンテスト（競争）** によりパフォーマンスを向上させるメカニズムを提案。全会一致型やマネージャー決定型とは異なる第3のアプローチとして、エージェントが互いに競争し、最も優れた判断を採用する設計を導入した。

#### HAD: Heterogeneous Multi-Agent Framework for Financial Sentiment Analysis (Xing 2024)
異種（Heterogeneous）マルチエージェントによるセンチメント分析に焦点を当て、同種のエージェントではなく、異なる能力・視点を持つエージェント群を組み合わせることでセンチメント分析の精度を向上させた。

## 3. 共通パターンと差異の分析

### 3.1 パイプライン型 vs 対話型

サーベイ対象の論文群は、エージェント間の協調プロトコルにおいて **パイプライン型（Sequential Pipeline）** と **対話型（Conversational/Debate）** の2つの基本アーキテクチャに大別される。

**パイプライン型** は MASFIN が代表例であり、各エージェントが明確に定義された順序で実行される。ステージ間のデータフローが一方向であるため、デバッグが容易で、各ステージの出力を個別に検証できる。投資判断のプロセスが「分解（Decomposition）」されているため、透明性が高く、人間のファンドマネージャーにとって理解しやすい。

一方、**対話型** は AlphaAgents が代表例であり、複数のエージェントがリアルタイムで議論を行い、相互にフィードバックしながら判断に到達する。FINCON の CVR もこのカテゴリに属し、フィードバックを通じた判断の改善が可能である。ただし、対話の収束保証がなく、計算コスト（LLM コール数）が予測困難である。

**ハイブリッドアプローチの台頭**: 実際には、多くのシステムが両方の要素を組み合わせている。MAS4InvestmentTeam の設計は、Phase 0-2（Postmortem, Screening, Analysis）をパイプライン型で実行し、Phase 3（Debate）を対話型で実行し、Phase 4-5（Decision, Portfolio）を再びパイプライン型に戻すハイブリッド構造を採用している。これはパイプラインの透明性と対話の判断品質向上を両立する設計である。

### 3.2 シングル LLM vs マルチ LLM

初期の研究（TradingGPT, Alpha-GPT）は単一の LLM（GPT-4）をベースに、プロンプトの差別化でエージェントの専門性を実現していた。近年の研究では、異なる LLM を異なるエージェントに割り当てる **マルチ LLM アーキテクチャ** が探索されている。

HAD の異種マルチエージェントフレームワークは、異なるモデル（例: GPT-4o をファンダメンタル分析に、Claude をセンチメント分析に）を割り当てることで、各モデルの強みを活かす可能性を示唆している。ただし、モデル間の出力フォーマットの不一致やコスト管理の複雑化といった実装上の課題がある。

当プロジェクトは Claude Code Agent Teams を統一的に使用するシングル LLM アーキテクチャを採用しているが、将来的にはファンダメンタル分析に長いコンテキストウィンドウを持つモデルを使い、スクリーニングにはより安価で高速なモデルを使うといった最適化の余地がある。

### 3.3 ディベート vs コンセンサス vs マネージャー決定型

エージェント間の意見集約メカニズムは、論文ごとに大きく異なる。

| メカニズム | 代表例 | 長所 | 短所 |
|-----------|--------|------|------|
| **全会一致型コンセンサス** | AlphaAgents | 安全性が高い、ハルシネーション抑制 | 凡庸な判断に収束、コントラリアンポジション困難 |
| **マネージャー決定型** | MAS4InvestmentTeam | 大胆な判断が可能、実際のファンド構造を再現 | マネージャーエージェントの品質に依存 |
| **内部コンテスト型** | Internal Contest (2025) | 競争によるパフォーマンス向上 | 最適解の保証なし |
| **概念的言語強化** | FINCON | フィードバックによる漸進的改善 | 大量の過去事例が必要 |

当プロジェクトがマネージャー決定型を採用した理由は、(1) 全会一致はリスク回避バイアスを増幅する、(2) ディベートログが完全な監査トレースを提供する、(3) 実際のファンド運用チームの意思決定構造を再現する、の3点である。ディベート自体は AlphaAgents と TradingAgents（UCLA/MIT）の Bull/Bear Researcher 構造から着想を得ているが、最終判断は Fund Manager エージェントに一任する点が異なる。

### 3.4 データソースの統合戦略

論文間で共通するのは、**構造化データと非構造化データの統合** が MAS の本質的な価値であるという認識である。

- **MASFIN**: OHLCV + SEC Filings + ニュース要約
- **AlphaAgents**: 財務指標 + アナリストレポート + センチメントスコア
- **HKUST**: 価格データ + LLM 生成ファクター
- **StockBench**: 市場データ + ニュースヘッドライン

当プロジェクトの PoC（Phase 1）では SEC Edgar tools + yfinance に限定しているが、Phase 2 以降で FRED マクロ指標、決算トランスクリプト、ニュースセンチメントを追加する計画は、この学術的知見と整合している。

## 4. 先読みバイアスと汚染対策の学術的知見

### 4.1 問題の本質: LLM の「未来知識」

MAS を用いたバックテストにおける先読みバイアスは、従来のクオンツバックテストとは質的に異なる問題を提起する。従来のバイアスは「データの不正確なタイムスタンプ」や「ルックアヘッドのあるテクニカル指標」といった明確な原因に帰着できた。しかし、LLM を用いる場合、**モデル自体が学習データを通じて「未来の知識」を保持している** という、根本的に新しいバイアスソースが発生する。

MASFIN が指摘するように、後知恵バイアス（Hindsight Bias）は過去の出来事を「予測可能であった」と誤認させる。2023年のバックテストを行う際に、GPT-4o が2024年の企業倒産を既に知っている場合、その知識が「リスク分析」として暗黙的に推論に漏洩する。この種の漏洩は、明示的なデータアクセスの制限だけでは防止できない。

### 4.2 StockBench の汚染チェック手法

StockBench が提案した汚染チェック手法は、この問題に対する最も実用的なアプローチである。手法は以下の3ステップで構成される。

1. **汚染テスト**: LLM に特定期間・特定銘柄の株価推移を予測させる。「AAPL は2023年Q3にどのような株価推移を見せたか？」のような質問を多数投げかける。
2. **汚染度の定量化**: 予測の正解率をランダムベースラインと比較し、統計的に有意に高い期間・銘柄を特定する。
3. **評価対象からの除外**: 汚染が検出された期間は、バックテストの評価対象から除外するか、結果の解釈において注意を付記する。

この手法は「LLM が何を知っているか」を直接的に測定するものであり、カットオフ日の設定という間接的な方法よりも堅牢である。

### 4.3 多層防御戦略

サーベイ対象の論文群から導かれる最良の実践は、**単一の対策ではなく多層防御** で先読みバイアスに対処することである。

**層1: データレベルの時間的隔離（Physical Isolation）**
- Point-in-Time データ管理: filing_date ベースのフィルタリング
- RAG/ベクトル DB のメタデータフィルタリング
- 訓練・検証・テストの時系列分離（HKUST の TS/VS/Test）

**層2: プロンプトレベルの時間的制約（Logical Constraint）**
- 時間的ペルソナの設定（「あなたは20XX年XX月XX日にいる」）
- 将来知識の使用を明示的に禁止するメタインストラクション
- Contemporaneous Data Snapshot（同時性のスナップショット）の強制

**層3: モデルレベルの汚染遮断（Model-level Decontamination）**
- ティッカー匿名化（企業名、CEO 名、ティッカーシンボルの置換）
- StockBench 式の汚染チェックと汚染期間の除外
- ニュースヘッドラインのデバイアス処理（結果を示唆する単語の中立化）

**層4: 事後検証（Post-hoc Validation）**
- Human-in-the-Loop によるエージェント推論のレビュー
- IC（Information Coefficient）の異常値検出
- Rolling Sharpe の安定性検証

当プロジェクトの3層バイアス防御（PoiT データ管理 + 時間的制約プロンプト + ティッカー匿名化）はこの学術的フレームワークの層1〜3をカバーしている。Phase 2 で予定されている汚染検出（Contamination Detection）は層3の StockBench 手法に相当し、Human-in-the-Loop レビューは層4に対応する。

### 4.4 生存者バイアスの排除

MASFIN の Postmortem Crew は、生存者バイアスの排除において独自のアプローチを示した。従来のクオンツ手法では、上場廃止銘柄をユニバースに含めるだけで対処していたが、MASFIN は倒産・上場廃止に至った企業の **「失敗パターン」を能動的に学習** し、スクリーニングフィルターとして活用する。

これは「除外」から「学習」への転換であり、単に生存者バイアスを統計的に補正するだけでなく、**失敗の予兆を検出する能動的なリスク管理機能** を付加している。当プロジェクトの Phase 0（Postmortem）はこの設計を直接的に踏襲している。

### 5.2 HKUST の Dynamic Strategy Optimization の Phase 1 スクリーニングへの応用

HKUST の研究が提示した3層 MLP による Dynamic Strategy Optimization は、当プロジェクトの Phase 1（Screening）に対して2つの重要な示唆を与える。

**第1の示唆: ファクターウェイトの動的調整**

現在の設計では、スクリーニングファクターのウェイトは固定値（momentum: 0.40, size: 0.20, quality: 0.25, value: 0.15）である。HKUST の手法を応用すれば、市場レジーム（強気・弱気・中立）に応じてファクターウェイトを動的に調整する **Weight Gating メカニズム** を導入できる。

例えば、ボラティリティが低い強気相場ではモメンタムファクターのウェイトを増加させ、高ボラティリティの弱気相場ではクオリティファクターを重視するといった適応が可能になる。Phase 3 で予定されている「動的ファクターウェイト調整（レジーム連動）」は、まさにこの HKUST 知見の応用である。

**第2の示唆: LLM 生成ファクターの追加**

HKUST は Cross-Section Operators と Time-Series Operators の組み合わせで LLM にファクター候補を生成させた。SSE50 での 53.17% の累積リターン（市場 -11.73%）は、LLM が人間の想定外のファクターを発見できることを示唆している。

当プロジェクトへの応用としては、Phase 1 のスクリーニングにおいて、従来の4ファクター（momentum, size, quality, value）に加えて、**LLM が SEC Filings のテキストから抽出した「テキストベースファクター」** を追加することが考えられる。例えば、10-K の MD&A セクションにおける特定キーワードの出現頻度やトーンの変化をファクター化する手法は、HKUST のフレームワークと SEC Filings の組み合わせで実現可能である。

ただし、これは Phase 3 以降のスコープであり、PoC では固定ファクターウェイトによるシンプルなスクリーニングで技術的な検証を優先する。

### 5.3 FINCON の Conceptual Verbal Reinforcement のディベートへの統合

FINCON の CVR（概念的言語強化）は、当プロジェクトの Phase 3（Debate）と Phase 4（Decision）に対して直接的な応用可能性を持つ。

**ディベート品質の漸進的改善**: CVR の核心は「過去の判断から学習する」ことにある。当プロジェクトでは、四半期ごとのリバランス（40-80回のバックテスト）を通じて、前期のディベートと実際のリターンの関係を分析し、次期のディベートのコンテキストとして注入することが可能である。

具体的には、以下のフィードバックループを構築できる。

```
Quarter Q のディベート → Quarter Q のポートフォリオ → Quarter Q+1 のリターン実績
    ↓
Quarter Q+1 のディベート開始時に以下を注入:
  - 「前期、Bull Advocate が主張した X は Y% のリターンをもたらした」
  - 「前期、Bear Advocate が警告した Z リスクは実際に顕在化した/しなかった」
```

ただし、**バックテストにおける CVR の適用には注意が必要** である。過去のリターン実績をフィードバックとして使用すること自体が先読みバイアスの一形態となりうる。この問題を回避するため、フィードバックは「T-2 四半期の実績」のみを「T 四半期のディベート」に使用する（1四半期のラグを設ける）設計が推奨される。

FINCON の CVR を当プロジェクトに導入する場合の段階的アプローチは以下の通りである。

1. **Stage 4（Phase 2 フル機能）**: ディベートログの構造化記録を開始。各ラウンドの主張と証拠を JSON で保存。
2. **Stage 5（バックテスト + 暗黙知拡充ループ）**: 過去のディベート結果と実績リターンの対応分析を実施。
3. **将来拡張**: CVR フィードバックループの導入。前期の「成功/失敗した論拠パターン」を次期のディベートコンテキストに注入。

### 5.4 TradingGPT の Layered Memory の当プロジェクトへの適用

TradingGPT の Layered Memory（短期・中期・長期の3層メモリ）は、当プロジェクトにおいて **KB（ナレッジベース）とリバランス間のコンテキスト引き継ぎ** という形で適用可能である。

**現在の設計の課題**: 現行の設計では、各リバランス日は独立したパイプライン実行であり、前期の判断コンテキストは引き継がれない。これは、人間のファンドマネージャーが前期の判断を覚えていて次期の判断に反映する実務とは乖離している。

**TradingGPT 知見の適用案**:

| メモリ層 | TradingGPT での用途 | 当プロジェクトでの適用 |
|---------|-------------------|---------------------|
| **短期メモリ** | 直近の取引結果と市場反応 | 前期リバランスからの変更内容と理由（`rebalance_trades.json`） |
| **中期メモリ** | 数週間のトレンドとパターン | 過去2-4四半期の conviction score の推移と実績リターン |
| **長期メモリ** | 投資哲学とマクロ的な教訓 | **dogma.md + KB1/KB2/KB3**（これは既に設計済み） |

注目すべきは、当プロジェクトの KB アーキテクチャが TradingGPT の「長期メモリ」に該当するものを **既に外部化・構造化して設計している** 点である。dogma.md（投資哲学）、KB1（8ルール）、KB2（12パターン）、KB3（5社34件の具体例）は、まさに「長期的な投資判断の基盤」として機能する。

TradingGPT が示した知見のうち、当プロジェクトにまだ反映されていないのは **短期・中期メモリ** の部分である。これを実装するには、各リバランス日の `decision_log.json` と `portfolio.json` を次期のパイプライン実行時にコンテキストとして注入する仕組みが必要となる。Phase 2 以降で以下のような構造を検討できる。

```
Phase 2（Analysis）開始時に注入するコンテキスト:
{
  "short_term_memory": {
    "previous_rebalance": "2025-03-31",
    "key_changes": ["AAPL: 4.2% → 3.1% (減少理由: バリュエーション懸念)", ...],
    "actual_performance": "+2.3% vs SPY +1.8%"
  },
  "medium_term_memory": {
    "rolling_4q_hit_rate": 0.62,
    "consistently_overweight_sectors": ["Technology", "Healthcare"],
    "repeated_mistakes": ["過去3期連続でエネルギーセクターを過小評価"]
  }
}
```

### 5.5 学術的知見に基づくアルファ探索自動化の方向性

Alpha-GPT、HKUST、QuantAgent の3つの研究は、それぞれ異なるアプローチでアルファ探索の自動化を追求している。これらの知見を統合すると、当プロジェクトの将来的なアルファ探索の方向性として以下の3つの軸が浮かび上がる。

**軸1: KB 駆動のアルファ仮説生成（Alpha-GPT の応用）**

Alpha-GPT の Human-in-the-Loop を KB に置き換える。具体的には、dogma.md の投資哲学（「競争優位性は能力であり結果ではない」等）から、LLM がアルファ仮説を自動生成する。例えば、「rule01（能力 ≠ 結果）を満たす企業は、結果のみで評価される企業よりも長期リターンが高い」という仮説を立て、これをバックテストで検証するサイクルを自動化できる。

**軸2: SEC Filings テキストからのファクター生成（HKUST の応用）**

HKUST の Cross-Section/Time-Series Operators を SEC Filings のテキストに適用する。10-K の MD&A セクションにおける「competitive advantage」「moat」「switching cost」等のキーワードの出現頻度・トーン変化をファクター化し、従来の財務ファクターに加える。当プロジェクトの edgar パッケージ（`EdgarFetcher`, `SectionExtractor`）がこのデータ取得の基盤として機能する。

**軸3: 判断品質の自動改善ループ（QuantAgent + FINCON の応用）**

QuantAgent のリサーチループ自動化と FINCON の CVR を組み合わせ、「バックテスト → 判断品質の評価 → KB の更新 → 再バックテスト」のサイクルを構築する。これにより、KB 自体がバックテスト結果に基づいて漸進的に改善される。ただし、このループにおける「先読みバイアス」の混入には細心の注意が必要であり、KB の更新は forward-looking な方法（将来のデータに基づかない修正）に限定しなければならない。

### 5.6 当プロジェクトの KB アプローチの学術的位置づけ

当プロジェクトの最も独自性の高い設計要素は、**KB（ナレッジベース）による投資哲学の注入** である。サーベイ対象の論文群において、投資哲学やドメインエキスパートの暗黙知を構造化してエージェントに注入するアプローチは **学術論文には存在しない**。

この独自性を学術的に位置づけると、以下のようになる。

**Alpha-GPT との比較**: Alpha-GPT は Human-in-the-Loop で人間の洞察をリアルタイムに翻訳するが、当プロジェクトの KB は人間の洞察を **オフラインで構造化・バージョン管理** している。Alpha-GPT のアプローチは人間が常に関与する必要があるのに対し、KB アプローチは一度構築すれば自律的に動作する。ただし、KB の鮮度維持には定期的な人間のレビューが必要であり、この点では完全な自動化ではない。

**FINCON との比較**: FINCON の CVR は「過去の判断結果からの学習」であり、経験的なフィードバックに基づく。当プロジェクトの KB は「投資哲学からの演繹」であり、原理原則からの演繹的チェックに基づく。CVR が帰納的アプローチであるのに対し、KB は演繹的アプローチであり、両者は相補的に機能しうる。

**TradingGPT との比較**: TradingGPT の Layered Memory は主に「過去の市場経験」の蓄積であるのに対し、当プロジェクトの KB は「投資の判断基準そのもの」の蓄積である。TradingGPT のメモリが「何が起こったか」を記憶するのに対し、KB は「何を見るべきか」を定義する。

**学術的な新規性の整理**:

| 側面 | 既存の学術的アプローチ | 当プロジェクトの KB アプローチ |
|------|---------------------|---------------------------|
| 知識の源泉 | LLM の事前学習データ + リアルタイム検索 | 人間のドメインエキスパートの体系化された暗黙知 |
| 知識の形式 | 非構造的（プロンプト内の自然言語） | 4層構造（dogma → KB1 → KB2 → KB3） |
| 知識の更新 | 自動（CVR、リサーチループ） | 半自動（人間レビュー + AI 提案） |
| 判断の一貫性 | モデル依存（temperature、シード） | KB により原則レベルで保証 |
| 透明性 | 推論チェーンの記録 | ルール適用結果の明示的記録（`rules_applied`） |
| 過信の抑制 | なし（多くのMASでAIは過信する傾向） | 確信度スケール（50% が最頻値、90% は極めてまれ） |

特に **「過信の抑制」** は学術論文では見過ごされがちなテーマである。多くの MAS 研究において、LLM エージェントは高い確信度で判断を下す傾向があるが、当プロジェクトの KB（特に dogma の確信度スケール）は「50% が最頻値であり、90% は極めてまれ」という人間の判断分布を AI に強制することで、過信を構造的に抑制する。これは AlphaAgents の全会一致型コンセンサスとは異なるアプローチで同様の目的（判断の慎重さの確保）を達成するものである。

### 5.7 フレームワーク比較論文からの実装指針

Joshi (2025) の "Architectures and Challenges of AI Multi-Agent Frameworks" は、CrewAI、LangChain、FinRobot 等のフレームワークを比較分析し、以下の将来方向を示唆している。

> **将来のハイブリッド**: CrewAI の協調 + LangChain のツール柔軟性 + FinRobot のドメイン特化

当プロジェクトは Claude Code Agent Teams を使用しており、これは上記のハイブリッドの一形態と位置づけられる。Agent Teams の協調プロトコル（チームリーダー → チームメイトの委任パターン）は CrewAI の思想に近く、MCP（Model Context Protocol）によるツール接続は LangChain のツール柔軟性に相当し、KB + SEC EDGAR MCP の組み合わせは FinRobot のドメイン特化に対応する。

Joshi が指摘する **パフォーマンスベンチマークと規制コンプライアンスの gap** は、当プロジェクトにとっても重要な課題である。特に、MAS が生成する投資判断が金融規制（投資助言業の登録要件、適合性原則等）にどう位置づけられるかは、学術研究ではほとんど触れられておらず、実用化においては避けて通れない論点である。当プロジェクトの `decision_log.json` による全判断の構造化記録は、規制コンプライアンスの観点からも重要な設計要素である。

## 6. 総括と今後の研究方向

### 6.1 サーベイからの主要な教訓

11本の論文と2本のサーベイ論文の分析から、以下の5つの主要な教訓が得られた。

1. **パイプライン型の透明性は投資判断において不可欠**: MASFIN の5段階パイプラインが実証したように、各ステージの入出力を構造化して記録することで、人間による検証と規制対応が可能になる。対話型のディベートは判断品質を向上させるが、パイプラインの中に明確な位置として組み込むべきである。

2. **先読みバイアスの排除には多層防御が必要**: StockBench の汚染チェック、MASFIN の生存者バイアス排除、時間的隔離の3つを組み合わせた多層防御が、学術的に最も堅牢なアプローチである。

3. **全会一致型コンセンサスは投資には不向き**: AlphaAgents の実験結果が示すように、全会一致は安全バイアスを増幅し、コントラリアンなポジションの取得を困難にする。マネージャー決定型が実務により適合する。

4. **LLM によるアルファ生成は有望だが、従来の検証プロセスとの統合が必要**: HKUST の SSE50 での好成績は印象的だが、LLM の出力をそのまま使うのではなく、IC 測定やバックテストによる統計的検証を経由させることが重要である。

5. **投資哲学の注入は学術的に未開拓な研究領域**: 当プロジェクトの KB アプローチは、既存の学術研究にはないユニークな貢献であり、Alpha-GPT の Human-in-the-Loop と FINCON の CVR の間に位置する新しいアプローチとして学術的にも価値がある。

### 6.2 当プロジェクトへの優先的な適用項目

| 優先度 | 適用項目 | 参照元 | 適用フェーズ |
|--------|---------|-------|------------|
| 高 | 3層バイアス防御の実装 | StockBench, MASFIN | Phase 1 MVP |
| 高 | Postmortem による生存者バイアス排除 | MASFIN | Phase 1 MVP |
| 高 | KB による過信抑制（確信度スケール） | 独自 + Alpha-GPT | Stage 3（KB統合） |
| 中 | 短期・中期メモリのリバランス間引き継ぎ | TradingGPT | Phase 2 |
| 中 | CVR フィードバックループ | FINCON | Stage 5 |
| 中 | ディベート品質の漸進的改善 | FINCON + AlphaAgents | Phase 2 |
| 低 | 動的ファクターウェイト（Weight Gating） | HKUST | Phase 3 |
| 低 | LLM 生成テキストファクター | HKUST + Alpha-GPT | Phase 3 |
| 低 | 内部コンテスト型の探索 | Internal Contest (2025) | 将来研究 |

### 6.3 未解決の学術的課題

サーベイを通じて、以下の課題が学術的にも実務的にも未解決であることが明らかになった。

1. **MAS のパフォーマンスの統計的有意性**: MASFIN の8週間、HKUST の1年間は、いずれも統計的に十分な長さとは言い難い。10年以上のバックテスト結果を持つ MAS 研究は存在しない。当プロジェクトが10-20年のバックテストを実行する計画は、この gap を埋める可能性がある。

2. **エージェント数とパフォーマンスの関係**: エージェントを増やすほどパフォーマンスが向上するのか、それとも一定数を超えると「議論のノイズ」が増えて劣化するのかは、体系的に検証されていない。

3. **KB の汎化性能**: 当プロジェクトの KB は特定のアナリスト（Y）の暗黙知に基づいているが、この KB が Y の判断を超えた汎化能力を持つかは未検証である。KB3 の5社34件という事例数が十分かどうかも学術的に興味深い問題である。

4. **コスト効率**: MAS の運用コスト（LLM API コール数 × 単価）とパフォーマンス向上の費用対効果は、ほとんどの論文で議論されていない。当プロジェクトのコスト見積もり（Phase 1: $500-1,500、Phase 2: $2,000-5,000）は、この実務的な gap に対する具体的なデータを提供する。
